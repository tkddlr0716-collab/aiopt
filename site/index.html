배포 전에, 비용사고를 먼저 잡자.
AIOpt는 baseline 사용 로그를 기반으로 변경(모델/컨텍스트/리트라이/트래픽)을 시뮬레이션하고, 사고 가능성을 텍스트 경고 + Exit code 0/2/3로 반환합니다.
→ 사람은 로그를 “계속” 보는 게 아니라, PR에서 Summary/코멘트로 “필요할 때만” 보면 됩니다.
5분 체험 CI/PR 자동 노출 로컬 대시보드 Pricing

Quickstart (Guardrail)

$ npx aiopt install --force
$ npx aiopt guard --context-mult 1.2 --call-mult 10
$ echo $?   # 0 OK / 2 WARN / 3 FAIL
저장 파일: aiopt-output/guard-last.txt, aiopt-output/guard-last.json

Guard가 보는 변화

  • model/provider 변경
  • context/output 길이 변화
  • retries 증가(특히 high confidence)
  • traffic spike: --call-mult

출력(복붙 가능한 텍스트)

WARN: possible LLM cost accident
Summary: baseline=$X → candidate=$Y (Δ=$Z)
Impact (monthly est): +$...
Accident risk: Medium/High
Confidence: Low/Medium/High (reasons...)
Recommendation: ...
로컬에서 동작하고, CI에 붙이면 PR에서 바로 보입니다.

Optional: Local Dashboard

$ npx aiopt dashboard --port 3010
# open http://127.0.0.1:3010/
로컬(127.0.0.1) 바인딩. 마지막 Guard/Scan 결과를 한 화면에서 확인.

CI Summary

  • Actions Step Summary에 자동 출력
  • 로그 스크롤 없이 한눈에 확인

PR Comment

  • WARN/FAIL이면 PR 쓰레드에 코멘트
  • 리뷰어가 “대화창”에서 바로 봄

Merge Gate

  • exit=3을 머지 차단으로 사용
  • 비용사고를 “배포 전에” 끊기
